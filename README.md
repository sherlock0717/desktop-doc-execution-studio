# 桌面求职材料执行台

一个本地运行的桌面工作台原型，用来把求职材料整理成可查看、可编辑、可导出的解析结果。

当前版本定位为 **v0.1 本地桌面原型**。它适合演示主流程和验证产品方向，不是成熟商用产品。

## 当前支持能力

- 空白起始页：启动后不自动进入示例。
- 示例案例：点击“示例”后加载小何材料包。
- 用户材料导入：支持 `txt / md / docx / pdf`，其中 `docx / pdf` 会尝试抽取正文。
- 材料类型确认：解析前需要确认全部材料类型。
- 能力裁剪：系统会按当前材料判断可生成内容，宁可少生成，也不生成空壳文档。
- 简历 only：生成主稿简历、简历修改建议和解析摘要；不生成岗位理解、面试准备或练习报告。
- 简历 + JD：增加岗位理解卡和基础面试表达辅导。
- 简历 + JD + 面试记录/项目补充材料：支持面试练习题和综合分析报告。
- 工作稿：以简历工作稿为主对象，可保存编辑结果。
- 建议闭环：建议可写入工作稿，成功后从待处理建议中消失。
- 导出 ZIP：只导出真正生成成功、内容非空且对用户有价值的文件。
- 桌面壳：已接入 Tauri，可作为本地桌面应用运行。

## 本地运行

### 1. 启动后端

```powershell
cd D:\desktop-doc-execution-studio
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r app\engine\requirements.txt
python -m uvicorn app.engine.main:app --host 127.0.0.1 --port 8000
```

默认后端地址是 `http://127.0.0.1:8000`。

### 2. 启动前端

另开一个终端：

```powershell
cd D:\desktop-doc-execution-studio
npm install
npm run dev
```

浏览器访问 Vite 输出的本地地址，通常是 `http://127.0.0.1:5173`。

### 3. 启动桌面版

需要先准备 Rust、Node.js 和 Tauri 2 构建环境。

```powershell
cd D:\desktop-doc-execution-studio
npm install
npm run build
npm run tauri:dev
```

当前后端仍需单独启动。

## AI / 模型说明

当前原型默认面向本地模型服务。后端会检查模型是否可用；模型不可用时，解析、建议写入、面试题生成、评分和综合报告会受影响。

如果使用 Ollama，请确认本地服务已启动，并且已经拉取配置中的模型。

失败时界面会尽量给出人话提示。综合分析报告只有两种合法状态：生成成功显示正式报告；生成失败显示失败提示，不渲染占位雷达图。

## 已知限制

- 当前是单机本地原型，没有账号、多案例云同步或权限系统。
- Python 后端需要单独启动，尚未随 Tauri 安装包内置。
- 模型输出质量依赖本地模型服务和材料质量。
- `docx / pdf` 抽取结果受文件格式影响，复杂版式可能不稳定。
- 完整面试练习和综合分析报告需要简历、岗位说明，以及面试记录或项目补充材料。
- 导出 ZIP 是基础实现，后续可继续补正式模板和更细的文件结构。
